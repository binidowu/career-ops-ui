# Impeccable — Design Context

## Design Context

### Users

**Who:** A generalist job seeker — open to any type of role or industry. Not locked into a single archetype (engineer, PM, designer), so the interface must never assume a specific career vertical or impose a narrow professional identity.

**Context:** Usage varies widely — late-night deep-dives, quick daytime pipeline scans between tasks, focused weekend evaluation sessions. The tool must work equally well in bright daylight and dim rooms, during both 5-minute checks and 2-hour research sprints. System-adaptive theming is non-negotiable.

**Job to be done:** Maintain calm, focused control over a complex job search. Reduce the cognitive overhead of evaluating many opportunities so the user can make clear-headed decisions. The interface should make the user feel *organized and in command*, never overwhelmed or rushed.

### Brand Personality

**3-word personality:** Measured · Versatile · Precise

**Voice:** Quiet confidence. The tool doesn't shout, celebrate, or gamify. It presents information with the restraint of a well-edited publication — every element earns its place. Tone is direct without being cold, intelligent without being pretentious.

**Emotional goals:**
- **Primary:** Calm, focused control — "I know exactly where I stand"
- **Secondary:** Quiet competence — the tool itself feels like proof of good judgment
- **Tertiary:** Clarity of thought — decisions feel easier, not harder

### Aesthetic Direction

**Visual tone:** Refined editorial meets technical precision. Think: the typography and spacing of a quality magazine masthead, combined with the information density and quiet logic of a well-designed research interface. Not flashy, not minimal-to-a-fault — *considered*.

**References (conceptual, not direct copies):**
- The measured restraint of *Monocle* or *Works That Work* magazine layouts
- The information clarity of Bloomberg Terminal's newer design language
- The typographic confidence of Stripe's documentation
- The quiet authority of a well-organized research paper

**Anti-references (MUST NOT resemble):**
- Consumer-gamified UIs (badges, streaks, confetti, progress rings)
- Fintech dark dashboards (heavy gradients, glowing charts, crypto energy)
- Corporate enterprise software (SAP, Workday — gray, boxy, lifeless)
- Recruiter / HR software aesthetic (bright blues, stock photos, fake friendliness)

**Theme:** System-adaptive. Both light and dark modes, derived from OS preference. Neither mode is the "real" design — both must feel equally intentional.

**Palette direction:** New direction entirely. The existing teal `hsl(187, 74%, 32%)` + purple `hsl(270, 70%, 45%)` from the CV template is being replaced. The new palette should:
- Be built in OKLCH for perceptual uniformity
- Pass color-vision-safe checks (deuteranopia, protanopia, tritanopia)
- Avoid the cliché AI palette (cyan-on-dark, purple-to-blue gradients)
- Use tinted neutrals rather than pure gray
- Keep accent usage rare — the 60-30-10 rule, with accents at ≤10%

### Design Principles

1. **Earned attention.** Every visual element must justify its presence. No decorative noise, no "design for design's sake." If it doesn't help the user make a decision or understand their pipeline, remove it.

2. **Editorial restraint.** Typography and spacing do the heavy lifting. Strong type hierarchy, generous whitespace, and precise alignment create the sense of quality — not colors, shadows, or effects.

3. **Inclusive by default.** WCAG AAA contrast ratios. Color-vision-safe palette (information never conveyed by color alone). Reduced motion support via `prefers-reduced-motion`. These aren't accommodations — they're the baseline.

4. **System-native comfort.** Light and dark modes are first-class citizens, not afterthoughts. The interface should feel like it belongs on the user's system, adapting naturally rather than imposing a branded visual identity.

5. **Information over decoration.** Dense when needed, spacious when helpful. The interface should handle 740+ tracked applications without feeling cluttered, and 3 applications without feeling empty. Adapt to the data, not the other way around.

### Accessibility Requirements

| Requirement | Level | Notes |
|-------------|-------|-------|
| Contrast ratios | WCAG AAA (7:1 normal text, 4.5:1 large text) | Non-negotiable for all text |
| Color vision | Safe for deuteranopia, protanopia, tritanopia | Never convey meaning by color alone; always pair with shape, position, or label |
| Reduced motion | `prefers-reduced-motion: reduce` | Disable all non-essential animation; transitions collapse to instant state changes |
| Focus indicators | Visible, high-contrast | Custom focus styles that meet AAA contrast against both light and dark surfaces |
| Screen readers | Semantic HTML, aria-labels where needed | All interactive elements must be keyboard-accessible and announced correctly |

### Typography Direction

Avoid the existing Space Grotesk + DM Sans pairing for new UI work (both are on the reflex-reject list). The new type system should:
- Pair a distinctive display face with a refined reading face
- Support the editorial + technical tone — not overtly "designy," not sterile
- Have strong weight variation for hierarchy (at minimum: regular, medium, bold)
- Include tabular figures for data-heavy views (scores, dates, counts)

### What Success Looks Like

If someone sees this interface and thinks "this person has good taste and clear thinking," the design is working. If they think "oh, another AI dashboard" or "this looks like a job board," it's failed.
